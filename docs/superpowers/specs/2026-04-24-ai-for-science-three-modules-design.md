# AI for Science 三模块前端设计

## 概要

当前站点从两个独立功能模块扩展为三个独立功能模块，首页仅保留产品介绍与功能入口，不承载统一工作台。三个模块分别是：

- 图像识别
- 模板匹配
- 时序预测

整体仍保持前端静态原型定位，不接真实模型、不接后端、不做真实上传计算。页面骨架、命名和静态数据结构需要为后续模型接入保留清晰边界。

## 产品结构

### 路由

- `/`
  - 首页
- `/image-recognition`
  - 图像识别
- `/template-matching`
  - 模板匹配
- `/time-series-forecast`
  - 时序预测

### 导航顺序

- 首页
- 图像识别
- 模板匹配
- 时序预测

### 首页角色

首页只承担三件事：

- 说明 AI for Science 前端原型定位
- 展示三个独立功能入口
- 提示当前版本为静态展示，后续可替换为真实模型服务

首页不展示工具台式操作区，不把三个模块合并进单一工作面板。

## 模块设计

### 1. 图像识别

#### 定位

图像识别模块定位为通用科研图像识别原型，用于从科研图像中提取对象与关键坐标。默认示例场景为单摆实验，通过图像识别得到摆球中心、支点等关键点坐标。

该模块不是特定学科页面，不限定显微、材料或医学场景。单摆实验只是默认示例，用来清楚表达“通过图像识别得到坐标结果”。

#### 页面结构

- 页面说明区
  - 说明这是科研图像识别静态原型
- 输入区
  - 展示样本来源、识别目标、输出类型、实验备注
- 结果区
  - 展示识别对象、关键点坐标、整体置信度、结果说明
- 示例预览区
  - 展示静态样本图与轻量关键点标记

#### 输入区字段

- 样本来源
  - 默认示例：单摆实验帧图像 / 显微图像 / 实验装置照片
- 识别目标
  - 默认示例：摆球中心、支点、参考刻度
- 输出类型
  - 默认示例：关键点坐标 / 边界框摘要 / 目标标签
- 实验备注
  - 默认示例：用于后续轨迹分析与角位移计算

#### 结果区字段

- 识别对象
  - 例如：摆球、支点、刻度参考线
- 关键点坐标表
  - 列包含：名称、x、y、置信度
- 整体置信度
  - 例如：94%
- 结果说明
  - 例如：已提取摆球中心与支点坐标，可进一步推导摆长与偏角

#### 交互边界

- 采用“识别摘要页”方案
- 不做真实图片上传
- 不做复杂框选、拖拽或多目标交互
- 只保留轻量样本图占位与静态关键点标记

### 2. 模板匹配

#### 定位

原“参数匹配”模块整体重命名为“模板匹配”。该模块用于将输入样本特征与模板库进行静态匹配展示，不再强调参数调优语义。

#### 页面结构

- 页面说明区
- 输入区
- 结果区
- 候选模板列表区

#### 输入区字段

- 目标模板
- 输入样本特征
- 约束条件
- 匹配策略

#### 结果区字段

- 推荐模板
- 匹配度
- 候选模板列表
- 匹配说明

#### 命名要求

所有面向用户的“参数匹配”文案、导航、路由、入口按钮、测试断言都改为“模板匹配”。

### 3. 时序预测

#### 定位

时序预测模块维持现有通用科研预测页面定位，用于展示科研序列数据的静态预测结构。

#### 页面结构

- 页面说明区
- 输入区
- 趋势图占位区
- 指标摘要区
- 结果说明区

#### 保持不变的核心内容

- 时间窗口
- 预测步长
- 变量选择
- 趋势图占位
- 预测区间
- 摘要指标

## 统一设计原则

### 页面骨架

三个模块都采用一致的高层结构：

- 说明区
- 输入区
- 结果区

其中图像识别页额外包含样本预览区，但整体复杂度仍与另外两个模块保持同一层级。

### 风格

- 中文科研用户
- 浅色极简
- 留白充足
- 低饱和科学蓝灰色系
- 不做拥挤仪表盘

### 组件策略

- 共享布局、导航、标题、卡片等基础组件继续保留在 `src/components`
- 三个功能模块保持各自独立 feature 目录
- 不为了这次改动引入额外 UI 库
- 不做过度抽象，优先维持页面和 feature 边界清晰

## 代码结构调整

- 新增 `src/features/imageRecognition`
- 原 `src/features/parameterMatching` 重命名为 `src/features/templateMatching`
- 首页继续使用共享入口卡片组件
- 路由和导航同步增加图像识别并改写模板匹配命名

## 静态数据边界

- 图像识别页使用单摆实验作为默认静态示例
- 模板匹配和时序预测继续使用各自 feature 内静态数据文件
- 未来如果接入真实模型，只替换各 feature 内的 `data.ts` 或 service 层，不改动页面骨架

## 测试与验收

### 自动化测试

- 首页测试更新为三个功能入口
- 新增图像识别路由测试
- 原参数匹配测试整体改为模板匹配测试

### 验收标准

- 首页显示三个入口卡片，顺序为图像识别、模板匹配、时序预测
- 导航顺序与首页入口顺序一致
- `/image-recognition` 可访问
- `/template-matching` 可访问
- `/time-series-forecast` 可访问
- 图像识别页能展示样本预览、关键点坐标和结果说明
- 模板匹配页不再出现参数匹配命名
- 移动端下首页与三个模块页均无横向滚动
- 构建通过

## 非目标

- 不接真实模型
- 不接后端接口
- 不做真实上传
- 不做复杂检测框交互
- 不引入登录、权限、数据库、任务轮询

## 默认实现假设

- 继续使用 React + Vite + TypeScript
- 样式继续使用现有 CSS 分层方式
- 图像识别优先表达“提取坐标结果”，而不是复杂视觉检测
- 单摆实验只是默认示例，不限制后续扩展到其他科研图像场景
